## چرا KeyFix؟

وقتی موقع تایپ بین انگلیسی، فارسی، عربی و آلمانی جابه‌جا می‌شوید، به‌راحتی یادتان می‌رود زبان فعلی کیبورد چیست. می‌خواهید `hello` بنویسید، اما چون زبان فعال فارسی است، متن به `اثممخ` تبدیل می‌شود. KeyFix این‌گونه اشتباه‌ها را به‌صورت کاملاً محلی تشخیص می‌دهد و کمک می‌کند قبل از اینکه بخشی از جمله شوند اصلاحشان کنید.

علاوه بر این، KeyFix می‌تواند غلط‌های املایی معمولی را هم (وقتی زبان کیبورد درست است) اصلاح کند. این دو قابلیت کاملاً از هم جدا هستند و برنامه همیشه می‌داند کدام نوع اصلاح را پیشنهاد می‌دهد.

## زبان‌های پشتیبانی‌شده

- انگلیسی
- فارسی
- عربی
- آلمانی

نکته‌ی مهم: بعد از نصب KeyFix وارد **Settings** شوید و فقط زبان‌هایی را روشن نگه دارید که واقعاً استفاده می‌کنید. بقیه را خاموش کنید. این کار دقت تشخیص را بالا می‌برد و اصلاح‌های اضافه را کم می‌کند.

## قابلیت‌های نسخه ۰.۶.۰

- اجرای برنامه در System Tray ویندوز به‌همراه پنل تنظیمات فشرده
- راه‌اندازی اولیه برای انتخاب زبان‌هایی که واقعاً استفاده می‌کنید
- فعال/غیرفعال کردن هر زبان به‌صورت جداگانه
- سه حالت تشخیص: `AlertOnly`، `AlertAndSuggest` و `AutoSwitch`
- اصلاح کلمه‌ی قبلی بعد از Space (نه وسط تایپ)
- تشخیص مبتنی بر دیکشنریِ مرتب‌شده بر اساس فرکانس برای هر چهار زبان
- تصحیح املایی آفلاین و اختیاری (ایندکس از نوع SymSpell) که **به‌صورت پیش‌فرض خاموش** است
- موتور تصمیم‌گیری محافظه‌کار با حاشیه‌ی ابهام و کنترل `Conservative`/`Balanced`/`Aggressive`
- تشخیص توکن‌های محافظت‌شده (URL، ایمیل، مسیر، نسخه، شناسه‌های کد و…) برای جلوگیری از اصلاح اشتباه
- دیکشنری شخصیِ محلی با import/export و جفت‌های جایگزینی اختیاری
- جایگزینی سریع متن با Unicode و مسیر پشتیبانِ کلیپ‌بورد
- اجرای خودکار هنگام روشن‌شدن ویندوز (اختیاری)
- صدای هشدار پیش‌فرض ویندوز و امکان انتخاب فایل `.wav` سفارشی
- اعلان‌های ویندوز
- لیست برنامه‌های مستثنی برای ترمینال‌ها، مدیرهای رمز و برنامه‌های حساس
- تشخیص کاملاً محلی، بدون telemetry و بدون سرور راه دور

## دو نوع تصحیح

KeyFix دو مسئله را از هم جدا می‌کند و در **Settings** می‌توانید هرکدام را جداگانه کنترل کنید:

| تنظیم | چه چیزی را اصلاح می‌کند | مثال | پیش‌فرض |
| --- | --- | --- | --- |
| اصلاح تایپ با زبان اشتباه کیبورد | کلیدهای درست را با زبان اشتباه زده‌اید | `اثممخ` ← `hello` | روشن |
| اصلاح غلط‌های املایی معمولی | غلط واقعی وقتی زبان کیبورد درست است | `recieve` ← `receive` | **خاموش** |

تصحیح خودکارِ املایی به‌صورت پیش‌فرض خاموش است؛ فقط اگر خواستید آن را (و در صورت تمایل گزینه‌ی «اعمال خودکار» را) روشن کنید.

## حالت‌های تشخیص

| حالت | کاری که انجام می‌دهد |
| --- | --- |
| `AlertOnly` | فقط هشدار (صدا/اعلان) می‌دهد و هیچ متنی را تغییر نمی‌دهد. |
| `AlertAndSuggest` | هشدار می‌دهد و زبان/کلمه‌ی بهتر را پیشنهاد می‌کند. |
| `AutoSwitch` | بعد از Space کلمه را اصلاح می‌کند و زبان ورودی را عوض می‌کند. |

## روش کار

KeyFix فقط یک بافر کوتاه از چند کاراکتر آخر را در حافظه نگه می‌دارد. وقتی Space می‌زنید، کلمه‌ی قبلی را بررسی می‌کند. اگر زبان دیگری آشکارا محتمل‌تر باشد، می‌تواند کلمه‌ی اشتباه را پاک کند، متن درست را جایگزین کند و زبان کیبورد را عوض کند.

مثال زبان اشتباه:

```text
هدف: hello
زبان فعال: فارسی
تایپ‌شده: اثممخ
اصلاح‌شده: hello
```

مثال املایی (فقط وقتی تصحیح املایی روشن باشد):

```text
زبان فعال: انگلیسی
تایپ‌شده: recieve
پیشنهاد: receive
```

بافر بعد از Enter، Tab، زبان پشتیبانی‌نشده، برنامه‌های مستثنی و اصلاح خودکار پاک می‌شود.

## توکن‌های محافظت‌شده

برای جلوگیری از اصلاح اشتباه، KeyFix هر چیزی را که کلمه‌ی ساده نیست اصلاح نمی‌کند. توکن‌های محافظت‌شده شامل این‌ها هستند: URL، ایمیل، مسیر فایل، آپشن‌های خط فرمان (`--configuration`)، شماره‌ی نسخه (`v0.6.0`)، دامنه، هشتگ، منشن، شناسه‌های کد (`camelCase`، `PascalCase`، `snake_case`، `SCREAMING_SNAKE`)، سرنام‌ها، اعداد، توکن‌های ترکیبیِ حرف‌وعدد و ایموجی. ترمینال‌ها و مدیرهای رمز هم به‌طور کامل از طریق لیست برنامه‌های مستثنی کنار گذاشته می‌شوند.

## دیکشنری شخصی

می‌توانید یک دیکشنری شخصیِ محلی از کلمات خودتان داشته باشید. کلماتی که اضافه می‌کنید همیشه معتبر در نظر گرفته می‌شوند و هرگز «اصلاح» نمی‌شوند، و می‌توانید جفت‌های جایگزینی هم تعریف کنید (مثلاً یک مخفف که به شکل کامل باز شود). این دیکشنری از add، remove، list، import (متن UTF-8) و export پشتیبانی می‌کند و به‌صورت محلی در این مسیر ذخیره می‌شود:

```text
%APPDATA%\KeyFix\user-dictionary.json
```

## حساسیت تصحیح

یک تنظیم واحد به نام «How eager» تعیین می‌کند KeyFix چقدر باید مطمئن باشد تا عمل کند:

| سطح | رفتار |
| --- | --- |
| `Conservative` | فقط وقتی اطمینان خیلی بالا و بدون ابهام است. پیش‌فرض. |
| `Balanced` | تعادل میان گرفتنِ اشتباه‌ها و پرهیز از اصلاح غلط. |
| `Aggressive` | اصلاحِ سریع‌تر؛ گرفتنِ بیشتر با کمی ریسک بالاتر. |

اصلاح خودکار همیشه نیاز دارد بهترین گزینه هم از آستانه‌ی اطمینان بگذرد **و** با حاشیه‌ای روشن از گزینه‌ی دوم جلوتر باشد؛ بنابراین موارد مبهم هرگز به‌صورت خودکار اصلاح نمی‌شوند.

## نصب

آخرین فایل نصبی را از صفحه‌ی [Releases گیت‌هاب](https://github.com/miladateight/KeyFix/releases/latest) دانلود کنید:

```text
KeyFixSetup-0.6.0.exe
```

بعد از نصب:

1. KeyFix را از Start menu اجرا کنید.
2. از کنار ساعت ویندوز واردش شوید.
3. وارد **Settings** شوید.
4. فقط زبان‌هایی را که استفاده می‌کنید فعال کنید.
5. زبان‌های اضافی را خاموش کنید.
6. انتخاب کنید برنامه فقط هشدار بدهد، پیشنهاد بدهد، یا خودش زبان و متن را اصلاح کند.

## حریم خصوصی

KeyFix طوری طراحی شده که متن کاربر را ذخیره نکند.

- متن تایپ‌شده روی دیسک ذخیره نمی‌شود.
- متن تایپ‌شده به هیچ سروری ارسال نمی‌شود.
- هیچ telemetry، SDK تحلیلی یا سرور راه دوری وجود ندارد.
- فقط یک بافر کوتاه داخل حافظه برای تشخیص استفاده می‌شود.
- تنظیمات در `%APPDATA%\KeyFix\settings.json` ذخیره می‌شود.
- دیکشنری شخصی به‌صورت محلی در `%APPDATA%\KeyFix\user-dictionary.json` ذخیره می‌شود و هرگز آپلود نمی‌شود.
- لیست پیش‌فرضِ مستثنی شامل مدیرهای رمز و ترمینال‌هاست.

جزئیات بیشتر در [PRIVACY.md](PRIVACY.md).

## نیازمندی‌های توسعه

- Windows 10 یا Windows 11
- .NET 8 SDK
- اختیاری: Visual Studio 2022
- اختیاری: Inno Setup 6 برای ساخت installer

## ساخت و تست

```powershell
.\scripts\build.ps1
```

ساخت و تست دستی:

```powershell
dotnet build .\KeyboardLanguageGuard.sln --configuration Release
dotnet test .\KeyboardLanguageGuard.sln --configuration Release
```

## اجرای برنامه

```powershell
dotnet run --project .\src\KeyboardLanguageGuard.App\KeyboardLanguageGuard.App.csproj
```

## ساخت فایل نصبی

```powershell
.\scripts\package-installer.ps1
```

خروجی در این مسیر ساخته می‌شود:

```text
artifacts\installer\
```

در مراحل نصب لوگوی AT8 نمایش داده می‌شود. آیکون برنامه و فایل نصبی از آیکون KeyFix استفاده می‌کنند. داخل نصب هم از اشکان غریب بابت ایده‌ی اولیه تشکر شده است.

## برنامه‌ی آینده

مواردی که برای نسخه‌های بعدی برنامه‌ریزی شده‌اند (در ۰.۶.۰ وجود ندارند):

- undo یک‌مرحله‌ای برای اصلاح خودکار
- یادگیری محلی بر اساس اصلاح‌های پذیرفته/ردشده
- مدل زبانی سبک bigram برای امتیازدهی بهتر
- ناحیه‌ی تست تشخیصی داخل برنامه و لاگ تشخیصیِ محلیِ اختیاری
- harness ارزیابی آفلاین با precision/recall اندازه‌گیری‌شده
- پروفایل جدا برای هر برنامه
- ترجمه‌ی کامل رابط تنظیمات
- امضای دیجیتال فایل نصبی
