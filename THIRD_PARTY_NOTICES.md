# Third-Party Notices

KeyFix includes the following third-party data. No third-party runtime code (NuGet packages beyond
the .NET base libraries and the xUnit test framework) is bundled in the shipped application.

## Word frequency lists

The embedded word lists in `src/KeyboardLanguageGuard.Core/Resources/` (the most common words for
Persian, English, German, and Arabic, one word per line, ordered by frequency) are **derived**
data: they contain only word strings, ordered by frequency rank, filtered to each language's script
and normalized. They are regenerated by `scripts/extract_wordfreq.py` from the sources below. See
`data/sources.json` for the machine-readable manifest.

| Source | URL | License | Used for |
|---|---|---|---|
| hermitdave / FrequencyWords (OpenSubtitles 2016 + 2018) | https://github.com/hermitdave/FrequencyWords | CC BY-SA 4.0 | en, de, ar, fa |
| rspeer / wordfreq | https://github.com/rspeer/wordfreq | Code: MIT/Apache-2.0; bundled data aggregated from permissively-licensed corpora | en, de, ar (primary frequency order) |
| behnam / persian-words-frequency (Persian Wikipedia) | https://github.com/behnam/persian-words-frequency | CC BY-SA 3.0 (Wikipedia-derived) | fa |

Because part of the source data is licensed under **CC BY-SA**, the embedded derived word lists are
made available under the same terms, with attribution provided here. The lists were filtered (only
words of each language's script, normalized, deduplicated, and limited to the most frequent entries)
before being embedded.

### Regenerating the lists

```powershell
pip install wordfreq
python scripts/extract_wordfreq.py --target 30000
```

Regeneration requires network access to the source repositories above. Normal application builds do
**not** require network access — the generated lists are committed to the repository.

## Bigram context asset

`src/KeyboardLanguageGuard.Core/Resources/bigrams-en.txt` is a small, hand-authored list of common
English word pairs with illustrative counts, written by the project — not derived from any
third-party corpus. It is original work under the same MIT license as the repository. See
`data/sources.json` for the manifest entry. Persian, Arabic, and German have no bigram asset yet.
